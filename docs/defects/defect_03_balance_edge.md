# D-03 — Перевод блокируется при равенстве доступной сумме

| Поле | Значение |
|---|---|
| **ID** | D-03 |
| **Заголовок** | На границе доступного остатка перевод отклоняется, хотя должен проходить |
| **Серьёзность** | Major |
| **Приоритет** | High |
| **Тип** | Функциональный / граничное значение |
| **Окружение** | Windows 10, Chrome, сервис из папки `dist` |
| **Воспроизводимость** | 100% |
| **Связанный тест-кейс** | TC-05 |
| **Автотест** | `tests/test_defect_balance_edge.py` |

## Краткое описание

По инструкции перевод можно осуществить, если сумма с учётом комиссии **не превышает** доступный остаток. При точном равенстве (сумма + комиссия = доступно) приложение показывает ошибку «Недостаточно средств».

## Шаги воспроизведения

1. Запустить сервис из папки `dist`
2. Открыть `http://localhost:8000/?balance=110&reserved=0`
3. Нажать «Рубли»
4. Ввести номер карты `1111222233334444`
5. Ввести сумму `100`
6. Проверить комиссию и наличие кнопки «Перевести»

## Ожидаемый результат

- Комиссия: 10 ₽
- Итого: 100 + 10 = 110 ₽, что равно доступному остатку
- Кнопка «Перевести» отображается, ошибки нет

## Фактический результат

- Комиссия: 10 ₽
- Кнопка «Перевести» не показывается
- Отображается «Недостаточно средств на счёте»

## Возможная причина

В коде используется строгое сравнение «больше нуля» вместо «больше или равно нулю» при проверке остатка.

## Рекомендация по исправлению

Разрешить перевод, когда сумма с комиссией равна доступному остатку.

## Влияние на пользователя

Пользователь не может перевести сумму, которая полностью укладывается в лимит счёта.
