# D-04 — Неверный расчёт комиссии 10%

| Поле | Значение |
|---|---|
| **ID** | D-04 |
| **Заголовок** | Комиссия считается неверно — банк недополучает до 9 ₽ за операцию |
| **Серьёзность** | Major |
| **Приоритет** | High |
| **Тип** | Функциональный / расчёт |
| **Окружение** | Windows 10, Chrome, сервис из папки `dist` |
| **Воспроизводимость** | 100% |
| **Связанный тест-кейс** | Чек-лист, раздел «Комиссия» |
| **Автотест** | `tests/test_defect_fee_calc.py` |

## Краткое описание

По инструкции комиссия составляет 10% от суммы перевода и округляется вниз. Фактически приложение округляет не сам процент, а сумму до десятков рублей — из-за этого комиссия часто меньше ожидаемой.

## Шаги воспроизведения

1. Запустить сервис из папки `dist`
2. Открыть `http://localhost:8000/?balance=30000&reserved=0`
3. Нажать «Рубли», ввести карту `1111222233334444`
4. По очереди ввести суммы `99`, `110`, `999`
5. Сравнить отображаемую комиссию с ожидаемой

## Ожидаемый результат

| Сумма | Комиссия (10%, округление вниз) |
|---:|---:|
| 99 | 9 |
| 110 | 11 |
| 999 | 99 |

## Фактический результат

| Сумма | Ожидалось | Получено |
|---:|---:|---:|
| 99 | 9 | 0 |
| 110 | 11 | 10 |
| 999 | 99 | 90 |

## Возможная причина

В коде комиссия считается как «десятки рублей от суммы», а не как 10% с округлением вниз.

## Рекомендация по исправлению

Использовать формулу: комиссия = целая часть от (сумма × 0,1).

## Влияние на пользователя

Банк систематически получает меньшую комиссию. Для сумм меньше 100 ₽ комиссия вообще равна нулю.
