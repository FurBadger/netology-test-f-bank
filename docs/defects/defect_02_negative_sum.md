# D-02 — Принимается отрицательная сумма перевода

| Поле | Значение |
|---|---|
| **ID** | D-02 |
| **Заголовок** | Поле «Сумма перевода» сохраняет минус, кнопка «Перевести» становится активной |
| **Серьёзность** | Critical |
| **Приоритет** | Critical |
| **Тип** | Функциональный / бизнес-логика |
| **Окружение** | Windows 10, Chrome, сервис из папки `dist` |
| **Воспроизводимость** | 100% |
| **Связанный тест-кейс** | TC-04 |

## Краткое описание

При вводе отрицательной суммы знак «минус» сохраняется в поле. Комиссия тоже становится отрицательной, кнопка «Перевести» доступна, а после нажатия появляется сообщение о принятии перевода на отрицательную сумму.

## Шаги воспроизведения

1. Запустить сервис из папки `dist`
2. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
3. Нажать «Рубли»
4. Ввести номер карты `1111222233334444`
5. В поле «Сумма перевода» ввести `-100`
6. Нажать «Перевести»

## Ожидаемый результат

- Минус отбрасывается, в поле остаётся положительное число
- Либо кнопка «Перевести» не появляется и показывается ошибка валидации

## Фактический результат

- В поле остаётся `-100`
- Комиссия отображается как `-10 ₽`
- Кнопка «Перевести» активна
- Alert: «Перевод -100 ₽ на карту 1111222233334444 принят банком!»

## Возможная причина

В обработчике ввода суммы знак минус специально возвращается в значение поля. Из-за отрицательных значений проверка «хватает ли средств» ошибочно считает перевод допустимым.

## Рекомендация по исправлению

Не сохранять минус в поле суммы и дополнительно проверять, что сумма перевода больше нуля.

## Влияние на пользователя

Критический дефект: пользователь может инициировать перевод с отрицательной суммой, что недопустимо для банковского сервиса.
