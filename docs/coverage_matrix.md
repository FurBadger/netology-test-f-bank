# Подбор тестовых данных

Документ описывает, какие значения использовались при составлении тест-кейсов и автотестов.

## Поле «Номер карты»

| Ситуация | Пример | Что должно быть |
|---|---|---|
| Пустое поле | — | форма суммы скрыта |
| Буквы и символы | `abc!@#` | игнорируются |
| 15 цифр | `123456789012345` | форма суммы скрыта |
| **16 цифр** | `1111222233334444` | форма суммы видна |
| 17 цифр | `12345678901234567` | форма не должна открываться (D-01) |

Проверяемые границы: 15, **16**, 17 цифр.

## Поле «Сумма перевода»

| Ситуация | Пример | Что должно быть |
|---|---|---|
| Отрицательная сумма | `-100` | блокировка (D-02) |
| В пределах лимита | `100` | перевод доступен |
| Ровно на границе | `100` при остатке 110 | перевод доступен (D-03) |
| Выше лимита | `100000` | «Недостаточно средств» |

При `balance=110`, `reserved=0` (доступно 110 ₽):

| Сумма | Комиссия | Итого | Ожидание |
|---:|---:|---:|---|
| 99 | 9 | 108 | перевод разрешён |
| 100 | 10 | 110 | перевод разрешён |
| 101 | 10 | 111 | перевод запрещён |

## Комиссия 10%

Правильная формула: 10% от суммы, округление вниз.

Проверяемые значения для D-04: 99 → 9, 110 → 11, 999 → 99.

## Связь требований и тестов

| Требование | Где проверяется |
|---|---|
| Три счёта на главной | `test_basic_checks.py` |
| Параметры balance/reserved из URL | `test_basic_checks.py`, `test_defect_balance_edge.py` |
| Номер карты — 16 цифр | `test_defect_card_length.py` |
| Комиссия 10% | `test_defect_fee_calc.py` |
| Сумма + комиссия ≤ остаток | `test_defect_balance_edge.py` |
| Alert после перевода | `test_basic_checks.py` |
| Сумма не отрицательная | `test_defect_negative_sum.py` |
