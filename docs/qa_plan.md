# План тестирования F-Bank

## Объект тестирования

Учебный банковский сервис F-Bank для перевода рублей. Приложение распространяется как архив `dist.zip`, запускается через `python -m http.server` из папки `dist`.

## Что проверяем

- отображение трёх счетов на главной странице;
- сценарий перевода с рублёвого счёта: выбор счёта, ввод карты, суммы, расчёт комиссии;
- проверку доступного остатка;
- сообщение об успешном переводе (alert).

## Что не проверяем (по заданию)

- операции с долларовым и евро счётами;
- действия после нажатия «Перевести»;
- тестирование в разных браузерах, нагрузку, безопасность.

## Подход к тестированию

| Этап | Что делаем |
|---|---|
| Ручное тестирование | 5 тест-кейсов + чек-лист |
| Документирование | баг-репорты на найденные дефекты |
| Автоматизация | Selenium + pytest, один автотест на дефект |
| CI | GitHub Actions, сборка должна быть красной из-за падающих тестов |

## Окружение

- **Локально:** Windows 10, Python 3.12, Google Chrome
- **CI:** Ubuntu, Python 3.12, Chrome (через GitHub Actions)

## Критерии завершения

- Базовые тест-кейсы TC-01 и TC-02 проходят
- Найдено не менее 2 разных дефектов
- На каждый дефект оформлен баг-репорт и написан автотест
- CI-сборка падает на тестах, воспроизводящих дефекты
