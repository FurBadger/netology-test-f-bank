# Ручные тест-кейсы F-Bank

**Окружение:** Windows 10, Google Chrome, сервис запущен командой `python -m http.server 8000` из папки `dist`.

**Перед началом:** сервис доступен в браузере, всплывающие окна (alert) не блокируются.

---

## TC-01. Успешный перевод

| Поле | Значение |
|---|---|
| Цель | Проверить, что корректный перевод проходит до подтверждения |
| Тип | Позитивный |
| Приоритет | Высокий |

**Шаги:**
1. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
2. Нажать на карточку «Рубли»
3. В поле «Номер карты» ввести `1111222233334444`
4. В поле «Сумма перевода» ввести `100`
5. Убедиться, что комиссия равна 10 ₽
6. Нажать «Перевести»

**Ожидаемый результат:** появляется сообщение «Перевод 100 ₽ на карту 1111222233334444 принят банком!»

**Фактический результат:** совпадает с ожидаемым. Тест пройден.

---

## TC-02. Недостаточно средств

| Поле | Значение |
|---|---|
| Цель | Проверить сообщение об ошибке при сумме больше доступного остатка |
| Тип | Негативный |
| Приоритет | Высокий |

**Шаги:**
1. Открыть `http://localhost:8000/?balance=30000&reserved=20001` (доступно 9 999 ₽)
2. Нажать «Рубли»
3. Ввести номер карты `1111222233334444`
4. Ввести сумму `100000`

**Ожидаемый результат:** показано «Недостаточно средств на счете», кнопка «Перевести» не отображается.

**Фактический результат:** совпадает с ожидаемым. Тест пройден.

---

## TC-03. Номер карты длиннее 16 цифр

| Поле | Значение |
|---|---|
| Цель | По инструкции номер карты должен состоять из ровно 16 цифр |
| Тип | Негативный, граничное значение |
| Приоритет | Средний |
| Связанный дефект | [D-01](../defects/defect_01_card_length.md) |

**Шаги:**
1. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
2. Нажать «Рубли»
3. В поле «Номер карты» ввести `12345678901234567` (17 цифр)

**Ожидаемый результат:** в поле остаётся не больше 16 цифр, форма «Сумма перевода» не появляется.

**Фактический результат:** в поле сохраняются все 17 цифр (`1234 5678 9012 3456 7`), форма суммы открывается. Тест не пройден.

## TC-04. Отрицательная сумма перевода

| Поле | Значение |
|---|---|
| Цель | Проверить, что отрицательную сумму нельзя перевести |
| Тип | Негативный |
| Приоритет | Высокий |
| Связанный дефект | [D-02](../defects/defect_02_negative_sum.md) |

**Шаги:**
1. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
2. Нажать «Рубли», ввести карту `1111222233334444`
3. В поле «Сумма перевода» ввести `-100`
4. Проверить значение в поле и наличие кнопки «Перевести»
5. Нажать «Перевести», если кнопка доступна

**Ожидаемый результат:** минус не сохраняется в поле, либо кнопка «Перевести» не появляется.

**Фактический результат:** в поле остаётся `-100`, комиссия `-10 ₽`, кнопка активна. После нажатия — alert «Перевод -100 ₽ … принят банком!». Тест не пройден.

---

## TC-05. Перевод на границе доступной суммы

| Поле | Значение |
|---|---|
| Цель | Если сумма с комиссией не превышает доступный остаток, перевод должен быть возможен |
| Тип | Граничное значение |
| Приоритет | Высокий |
| Связанный дефект | [D-03](../defects/defect_03_balance_edge.md) |

**Шаги:**
1. Открыть `http://localhost:8000/?balance=110&reserved=0` (доступно 110 ₽)
2. Нажать «Рубли», ввести карту `1111222233334444`
3. Ввести сумму `100`
4. Проверить комиссию и наличие кнопки «Перевести»

**Ожидаемый результат:** комиссия 10 ₽, итого 110 ₽ — перевод разрешён, кнопка видна.

**Фактический результат:** комиссия 10 ₽, но показано «Недостаточно средств на счёте», кнопки нет. Тест не пройден.
